# 3-5 分钟演示脚本

## 0. 启动

本地启动：

```bash
cd backend
mvn spring-boot:run

cd frontend
npm install
npm run dev
```

打开 `http://localhost:5173`。

## 1. 介绍业务目标

“这个 Demo 模拟市场线索从投放进入系统后，经过外呼清洗、加微、销售跟进，最终转为 MQL 和 SQL。重点不是完整 CRM，而是验证核心业务闭环、数据建模、状态流转和 AI-coding 落地能力。”

## 2. 演示线索列表

- 打开“线索工作台”。
- 展示模拟数据。
- 按状态、来源或关键词筛选。
- 切换角色：主管看全部，销售张三/李四只看自己的线索。

## 3. 演示新增与导入

- 点击“新增线索”，填写姓名、手机号、来源、渠道。
- 点击“导入 CSV”，选择 `data/sample_leads.csv`。
- 说明：系统按手机号去重，会展示新增、重复、失败数量。

## 4. 演示核心流转

选择一条线索：

1. 分配销售负责人。
2. 修改状态为有效或用“三方回调”模拟外呼系统回传有效。
3. 修改状态为已加微或用企微/SCRM 回调模拟加微成功。
4. 添加一次跟进记录。
5. 标记为 MQL。
6. 再标记为 SQL。

强调：SQL 必须从 MQL 转化，且必须已有负责人；MQL 前必须有跟进记录。

## 5. 演示详情审计

- 点击线索行打开详情抽屉。
- 展示跟进记录。
- 展示状态历史。
- 展示操作日志。

## 6. 演示加分项

- 漏斗看板：展示各状态数量、有效率、加微率、MQL/SQL 转化率。
- 异常提醒：展示 48 小时未跟进的有效线索。
- 自然语言查询：输入“本周哪个渠道 SQL 转化率最高？”
- 三方回调：模拟外呼和企微/SCRM 回调。
- Docker：说明支持 `docker-compose up --build`。

## 7. 收尾说明

“下一步我会优先做真实权限、回调幂等和更完整的自然语言查询。当前版本保留了状态历史、操作日志和清晰 README，方便评审者快速启动和追问核心代码。”
