## 使用工具

- ChatGPT / Codex：需求拆解、状态机设计、接口草稿、前后端代码生成、测试和文档整理。
- Maven / npm：通过构建反馈发现 Java 版本、Maven 插件版本、Vue 模板和打包问题。
- 人工复核：重点检查权限边界、状态流转、自然语言查询语义、测试稳定性和文档真实性。

## AI 参与环节

- 需求拆解：将题目拆成线索进入、外呼清洗、加微承接、私域跟进、MQL/SQL、漏斗统计、异常提醒、三方回调和自然语言查询。
- 数据建模：生成线索、销售、跟进记录、状态历史、操作日志、导入批次和三方回调日志等对象。
- 接口实现：生成线索列表、新增、状态变更、负责人分配、跟进记录、CSV 导入、漏斗、异常、三方回调和自然语言查询 API。
- 前端页面：生成 Vue + Element Plus 工作台、看板、详情抽屉、异常页、查询页和回调页。
- 测试和交付：生成 README、schema、演示脚本、Docker Compose 和后端业务测试。

## AI 输出中的具体问题与修正

1. 角色权限最初是“前端假安全”。
   - 问题：前端通过角色切换传 `ownerId`，但后端没有真正根据当前用户角色限制数据，直接调接口可以绕过。
   - 修正：前端发送模拟登录头 `X-User-Id`；后端根据该用户的 `role` 强制覆盖查询条件。销售角色即使传 `ownerId=null` 也只能看到自己的线索，主管才可看全量。

2. 状态流转校验过宽。
   - 问题：初版只拦截少数非法路径，`NEW -> MQL`、`SQL -> NEW`、`WECHAT_FAILED -> MQL` 等跳转可能被放过。
   - 修正：在后端增加白名单式状态流转表，只允许文档声明的核心路径；同时保留 MQL 必须有跟进记录、SQL 必须有负责人等业务约束。

3. 自然语言查询的“本周”最初没有真实过滤。
   - 问题：用户问“本周哪个渠道 SQL 转化率最高”，初版返回的是全量历史渠道转化率。
   - 修正：识别“本周/this week”，按本周创建线索和本周 SQL 时间计算渠道 SQL 转化率。

4. 测试用例最初硬编码用户 ID。
   - 问题：测试里写死 `1L`，依赖初始化顺序，换环境或测试顺序变化可能失败。
   - 修正：测试通过 `SalesUserRepository` 动态查找销售用户，再执行分配和流转。

5. CSV header 识别过脆弱。
   - 问题：只识别 `phone` 或 `手机号`，如果 header 是“姓名,电话,来源,渠道”可能被当成数据。
   - 修正：扩展 header 检测，识别 `phone/mobile/tel/电话/手机/姓名,电话/姓名,手机` 等常见写法。

6. 手动创建线索也被标记为导入。
   - 问题：实体 `prePersist()` 会给所有新线索写 `importedAt`，语义不准确。
   - 修正：只在 CSV 导入时设置 `importedAt`，手动新增仅保留 `createdAt`。

7. 缺少并发状态变更保护。
   - 问题：状态变更是 read-check-write，两个操作者并发修改时可能互相覆盖。
   - 修正：在线索实体增加 `@Version` 乐观锁字段，数据库 schema 同步增加 `version`。

8. 前端初版单文件过重。
   - 问题：`App.vue` 承载多个 Tab、Dialog 和所有状态管理，不利于扩展。
   - 修正：拆出 `DashboardPanel`、`QueryPanel`、`AnomalyPanel`，保留 `App.vue` 做壳层和主流程协调。

9. 构建环境兼容问题。
   - 问题：Spring Boot 默认 Maven 插件要求 Maven 3.6.3，而本机 Maven 为 3.6.1。
   - 修正：在 `pom.xml` 中锁定 `maven-compiler-plugin` 和 `maven-jar-plugin` 到兼容版本。
